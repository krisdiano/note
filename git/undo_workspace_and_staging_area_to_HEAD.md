# 如何撤销工作区和暂存区到HEAD

有时你修改了工作区的文件且添加到了暂存区，在这之后可能也再一次修改了一些工作区文件。如果工作区和暂存区的所有修改是错误的，想要将所有修改恢复到HEAD版本，可以借助git reset命令。



## 解决命令

在”如何让暂存区恢复为HEAD“文章中介绍了git reset的这些方式，在这里用到了-hard参数。

```
git reset -hard HEAD
```



## 针对某个文件

在上面的基础上，考虑一下如何将暂存区和工作区的某个文件恢复到HEAD版本。

```
// 1：将暂存区恢复到HEAD版本
git reset HEAD <file>
// 2：将工作区恢复到暂存区
git checkout -- <file>
```
